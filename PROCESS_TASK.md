# Claude Scheduled Task — обработка заметок из Telegram

## Задача

Ты обрабатываешь входящие заметки из Telegram и создаёшь дневной план.
Рабочая директория: `/Users/vladimirgavrilow/Documents/test/telegramObsidian`

---

## Шаг 1 — Обработка inbox.json

1. Прочитай `inbox.json`
2. Если нет необработанных сообщений (`processed: false`) — переходи к Шагу 2
3. Для каждого необработанного сообщения:
   - Определи категорию по смыслу:
     - `health` — здоровье, самочувствие, лекарства, тренировки
     - `ideas` — идеи, мысли, концепции
     - `shopping` — покупки, списки товаров
     - `finance` — деньги, расходы, доходы
     - `books` — книги, статьи, материалы для чтения
     - `tasks` — дела, задачи, напоминания
     - `workout` — тренировки, упражнения, спорт
     - Новая категория — если ни одна не подходит
   - Если в тексте явно написана категория (например "здоровье: ...") — используй её
   - Дозапиши в `notes/<category>.md` в формате:

```markdown
## YYYY-MM-DD

- **Краткое название** — описание
  > Дополнительный комментарий или контекст (если есть что добавить)
```

   - Отметь запись как `processed: true` в inbox.json
4. Удали все записи с `processed: true` из inbox.json и сохрани файл

## Шаг 2 — Дневные планы

### Правило дат

Каждая запись в категориальных файлах (`tasks.md`, `workout.md`, `shopping.md` и др.) имеет раздел с датой (`## YYYY-MM-DD`).
- Запись под разделом с **сегодняшней датой** → только в сегодняшний план
- Запись под разделом с **завтрашней датой** или без конкретной даты → только в завтрашний план
- **Никогда не дублируй одну запись в оба плана**

### 2а. Сегодняшний план

Создай файл `notes/daily/<СЕГОДНЯ>.md` если его нет, или дозапиши в него недостающее. Не удаляй существующие пункты.

Если сегодня **будний день (пн–пт)** — добавь повторяющиеся задачи из `notes/recurring.md` раздела "Каждый будний день", если их ещё нет в файле.
Раздел "Каждый день" добавляй всегда, если ещё нет.

Если среди обработанных сообщений есть задачи с явным указанием "сегодня" — добавь их в нужный раздел.

### 2б. Завтрашний план

Создай или обнови файл `notes/daily/<ЗАВТРА>.md` (следующий день после сегодня).
Включай только записи с датой на завтра или без конкретной даты. Записи с сегодняшней датой — пропускай.

Если завтра **будний день (пн–пт)** — обязательно включи повторяющиеся задачи из `notes/recurring.md` раздела "Каждый будний день".
Раздел "Каждый день" из `notes/recurring.md` включай всегда.
Повторяющиеся задачи всегда добавляй в начало соответствующего раздела.

Формат файла:

```markdown
# План на YYYY-MM-DD (завтра)

## 🏋️ Тренировка
- [ ] Тренировка          ← из recurring.md (будни)
- [ ] (из workout.md — только записи с датой на завтра или без даты)

## ✅ Дела
- [0/4] Работа — 4 рабочих блока по ~1.5 часа   ← из recurring.md (будни)
- [ ] (из tasks.md — только записи с датой на завтра или без даты)

## 🛒 Покупки
- [ ] (из shopping.md — только записи без конкретной даты или с датой на завтра)

## 💡 Идеи для проработки
- (из ideas.md за последние 3 дня)

## 🔔 Напоминания
- (любые напоминания или дедлайны)
```

Заполняй только те разделы, для которых есть данные. Пустые разделы не включай.

## Важно

- Пиши по-русски
- Будь кратким, но информативным
- В комментариях (>) добавляй только полезный контекст, не пересказывай сообщение
- Если категория новая — создай файл `notes/<category>.md` с заголовком `# <Категория>`
