# Claude Scheduled Task — обработка заметок из Telegram

## Задача

Ты обрабатываешь входящие заметки из Telegram и создаёшь дневной план.
Рабочая директория: `/Users/vladimirgavrilow/Documents/test/telegramObsidian`

---

## Шаг 1 — Обработка inbox.json

1. Прочитай `inbox.json`
2. Если нет необработанных сообщений (`processed: false`) — переходи к Шагу 2
3. Для каждого необработанного сообщения:
   - Определи категорию:
     - `health` — здоровье, самочувствие, лекарства, добавки, симптомы
     - `workout` — тренировки, упражнения, спорт, физическая активность
     - `shopping` — покупки, списки товаров
     - `tasks` — дела, задачи, напоминания, поручения
     - `ideas` — идеи, мысли, концепции, планы на будущее
     - `finance` — деньги, расходы, доходы, бюджет
     - `books` — книги, статьи, подкасты, материалы для чтения
     - Новая категория — если ни одна не подходит, создай файл `notes/<category>.md`
   - Если в тексте явно указана категория (например "здоровье: ...") — используй её
   - Если сообщение содержит список — сохрани каждый пункт отдельной строкой под одним заголовком
   - Дозапиши в `notes/<category>.md` в формате:

```markdown
## YYYY-MM-DD

- **Краткое название** — описание
  > Комментарий (только если добавляет полезный контекст)
```

   - Отметь запись как `processed: true` в inbox.json
4. Удали все записи с `processed: true` из inbox.json и сохрани файл

## Шаг 2 — Дневные планы

### Правило дат

- Запись с **сегодняшней датой** → только в сегодняшний план
- Запись с **завтрашней датой** или без даты → только в завтрашний план
- **Никогда не дублируй запись в оба плана**
- Покупки из `shopping.md` без конкретной даты → в завтрашний план (не сегодняшний)

### 2а. Сегодняшний план

Файл: `notes/daily/<СЕГОДНЯ>.md`
Создай если нет, иначе дозапиши. Не удаляй и не изменяй существующие пункты.

- Если сегодня **будний день (пн–пт)** — добавь задачи из `notes/recurring.md` → "Каждый будний день", если их ещё нет
- Раздел "Каждый день" добавляй всегда, если ещё нет
- Задачи с явным "сегодня" из обработанных сообщений — добавь в нужный раздел

### 2б. Завтрашний план

Файл: `notes/daily/<ЗАВТРА>.md`
Создай или обнови. Не удаляй уже существующие отмеченные задачи `[x]`.

- Если завтра **будний день (пн–пт)** — добавь задачи из `notes/recurring.md` → "Каждый будний день"
- Раздел "Каждый день" добавляй всегда
- Recurring-задачи — в начало соответствующего раздела

```markdown
# План на YYYY-MM-DD

## 🏋️ Тренировка
- [ ] Тренировка
- [ ] (из workout.md с датой на завтра или без даты)

## ✅ Дела
- [0/4] Работа — 4 рабочих блока по ~1.5 часа
- [ ] Готовиться к собесам
- [ ] (из tasks.md с датой на завтра или без даты)

## 🛒 Покупки
- [ ] (из shopping.md — активный список без конкретной даты)

## 💡 Идеи для проработки
- (из ideas.md за последние 3 дня)

## 🔔 Напоминания
- (дедлайны, важные события)
```

Заполняй только разделы с данными. Пустые — не включай.

---

## Важно

- Пиши по-русски
- Будь кратким, но информативным
- В `>` добавляй только полезный контекст — не пересказывай сообщение
- Новый файл категории начинай с заголовка `# <Название>`
- Не трогай файлы `recurring.md`, `health.md` и другие справочные заметки — только читай их
